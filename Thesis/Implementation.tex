\chapter{Implementation}

In this chapter the implementational details of the concept presented in chapter \ref{Concept} are described. The software runs on a Linux operation system and uses ROS Kinetic as middle-ware. The usage of ROS allows easy communication between different processes, also called nodes, and the easy integration of existing packages. Existing packages are used for the communication with the robot and the sensors, and the source code of some packages was used as a starting point for the implementation. Also useful is the tf library, which is integrated into ROS. It keeps track of all coordinate frames and allows nodes to obtain the most recent transformation between two coordinate frames and also the transformations at past times. Other used libraries are OpenCV, PCL, the mapping libraries Octomap and Gmapping, and the CNN libraries Caffe and Darknet. OpenCV is an open-source image processing library. In this work it was not only used for image processing but also in the internal representation of grid maps and for operations executed on those maps. PCL (Point Cloud Library) was used for operations on point clouds, like coordinate transformations. The whole software was written in C++.

Figure \ref{ImplOverview} shows the structure of the implementation, with its nodes and the communication channels between the nodes. The following sections contain an in-depth description of the implementation of the nodes and the communication between the nodes.
