\section{Doorway Detection Node}

The input data for the doorway detection node is point clouds from the upward looking RGBD-camera and range measurements from the LIDAR. The most recent range measurements from the LIDAR are stored for further processing. When a new point cloud arrives and the robot has moved considerably since the last execution of the doorway detection, the doorway detection is started. The movement check is done because of performance reasons and to avoid redundant information in the mapping. The output is a set of doorway detections represented as an array of poses in the robot coordinate frame.

The first step is to transform the point cloud from the coordinate frame of the camera into the coordinate frame of the robot using the known static transformation. Then two subsets of points are extracted from the point cloud, one subset containing only points with a height between 1.95\,m and 2.05\,m and one subset containing only points lower than 1.95\,m. In a next step both subsets of points are projected onto the ground and converted into a 2D image, represented as OpenCV matrix. Those images are shown in figure \ref{fig:door detection precedure} (a) and (b). As can be seen in the left image not only points of the upper part of the door frame are at about 2\,m height, but also points on the walls. The purpose of the second image, which also contains those walls, is to eliminate the undesired pixels from the walls. Therefore the second image is subtracted from the first one. Applying the morphological operations closing - to fill possible holes - and opening - to remove stray pixels - results in the filtered image shown in figure \ref{fig:door detection precedure} (c).

The filtered image is searched for contours and around every found contour a the minimum area rectangle is fitted, both using OpenCV functions. This can be seen in figure \ref{door res}. All rectangles with $d<0.05\,m \wedge d>0.5\,m \wedge w<0.6\,m$ are ruled out. The width and depth can be converted from pixels to meters using the known number of pixels per meter.

To do the other checks described in section \ref{DoorwayDetection}, the latest laser scan from the LIDAR has to be transformed into the correct coordinate frame. Laser scan and point cloud are usually not acquired at the same time, so robot movement in the time between the measurements has also to be considered. This is done using the tf library, which allows to specify not only the coordinate frames when accessing a transformation but also the time of interest. Three areas are defined in figure \ref{door res}. For every area the number of points from the transformed laser scan within the area are counted. If the sum of points in the red areas is smaller 5 and the number of points in the green area is greater 30, then a doorway is found. In this case the pose of the doorway is estimated as described in section \ref{DoorwayDetection} and added to the result array. This is done for all rectangles and so the result array contains all detected doorways.
