\section{High-Level Planner}

This section covers the high-level planner, which operates on the topological map and selects the room the robot should search next. Based on the selected target room it also generates high-level tasks, which are then planned and executed by the low-level planner. The goal of the planner is to minimize the expected search time, which is a reasonable objective for object search. Other possibilities would be maximizing the probability of finding the object in limited time or minimizing the time until a certain confidence about the existence of a searched object is reached. These tackle slightly different problems, but the results should be quiet similar in most scenarios. The high-level planner is called when a search task is started and also every time a high-level tasks finishes. 


\subsection{High-Level Planning}

The objective of the high-level planner is to find the optimal search path $\mathbf{p^*}=(r_1,r_2,...,r_{\mathbf{|R|}})$ with $r_i$ being the $i^{th}$ room to search and $\mathbf{|R|}$ being the number of not searched rooms given a starting room $r_0$. A valid search path contains every not already searched room exactly once. In this work the optimal path is the valid path with the minimal expected search time.
\begin{equation}
 \mathbf{p^*}=\argminA_{\mathbf{p} \in valid paths} E(t|\mathbf{p},r_0) 
\end{equation}

The expected search time given a staring room $r_0$ and a search path $\mathbf{p}$ can be calculated with
\begin{equation}
 E(t|\mathbf{p},r_0) = \sum_{i=1}^{\mathbf{|p|}} (\prod_{j=1}^{i-1}(1-P_{r_j}(o))(M_{r_{i-1},r_i}+E_{r_i}) 
\end{equation}
This calculation is illustrated in an example calculation in Figure \ref{fig: search tree}. Searching rooms with high object probabilities first results in a lower expected search time as the probability that another room has to be searched is reduced. However the probability of finding the object is the same for all paths. The matrix $M$ contains estimations of the movement times, where $M_{r_i,r_j}$ is the time needed to drive from room $r_i$ to room $r_j$. The matrix elements are calculated using the movement times described at the end of section 5.5.7 and the topological map. 

The possible paths form a search tree, as illustrated in Figure \ref{fig: search tree}. The tree representation shows possibilities for optimizations in the calculation like the reuse of intermediate results and pruning.

\subsection{Generation of next High-Level task}

The search path contains only the order, in which the rooms are searched. To execute the search path tasks have to be generated, which are then planned and executed by the low-level planner. These high-level tasks are:
\begin{itemize}
 \item \textbf{Move through doorway:} A doorway is specified through which the robot should drive to enter another room.
 \item \textbf{Peek:} The robot should turn left and right to quickly gather some information about the room.
 \item \textbf{Explore room:} The robot should explore the current room to obtain knowledge about likely object locations and find all doorways in the room.
 \item \textbf{Search room:} The robot should search for the target object in the current room
\end{itemize}

Figure \ref{fig: HLTask} depicts the generation of the high-level tasks. Therefore the high-level planner keeps track of whether a room was visited, explored or searched. The topological map is used to get the next room to enter on the way to the room to search. As replanning is done after every high-level task only the next high-level task has to be generated.
