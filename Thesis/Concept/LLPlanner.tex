\section{Low-Level Planner}

The low-level planner's main task is to generate the goal poses the robot should drive to to execute the current high-level task. With the exception of the move through doorway task all those tasks are executed within a single room. Therefore the low-level planner can ignore other rooms to make the planning easier. In addition the low-level planner also uses the results of the object detection to decide if an object of the searched type is found. This section gives an overview on how those tasks are planned and executed.

\subsection{Move Through Doorway Task}

The goal of this task is to drive through a specified doorway into the next room. It is the only high-level task, where the robot can go into another room. The execution of this task is visualized in figure \ref{fig:move_trough_doorway}. In a first step the robot is sent to a position in front of the doorway facing towards the doorway. In this process the door pose becomes more accurate and possible obstacles behind the doorway are detected. In a second step the robot is sent to a goal behind the doorway. When the robot passes the doorway the map is switched. During the execution the low-level planner keeps track of the doorway pose and changes the goal pose accordingly. This is especially important when the map switch occurs as the doorway pose can be completely different in the new rooms map.

\subsection{Peek Task}

The purpose of this task is to quickly get a first impression of a room never visited before. This is achieved by turning the robot left and right, so more of the room is seen by the camera.

\subsection{Explore Room Task}

The goals of the explore room task is to find all doorways in the room, detect likely object locations and also to find easy to spot target objects. 2D exploration is done to achieve these goals. After finished 2D exploration all doorways are guaranteed to be found as space behind an undiscovered doorway would be unexplored. Also most of the room is seen at least vague, though some areas might be completely unseen due to the smaller field of view of the camera compared to the LIDAR. 

A frontier-based approach is used for exploration. In this thesis a frontier is defined as an accessible cell in the occupancy grid map adjacent to at least one unexplored cell, as shown in figure \ref{fig: exploration}. Based on an objective function a frontier is selected and the robot is sent to this frontier and therefore exploring the area beyond the frontier. In this work the objective function is the distance between the robot and the frontier, where nearer frontiers are preferred.

 \caption{A map with accessible cells (purple and yellow cells) and frontiers (yellow cells)}

Only the current room should be explored. To prevent the robot from driving out of the room during exploration all detected doorways are blocked by virtual obstacles as shown in figure \ref{fig: exploration_door_blocked}. 

If the object detection detects an object with a sufficiently high confidence the object is assumed to be found. The estimated location in space is reported and the search is stopped. A decision based on multiple images, like during the search task, is not done during the exploration.

\subsection{Search Room Task}

The goal of the search room task is to find an object of the searched type within the current room or to fully search the current room if no searched object is in the room. The low-level planner has to find the optimal next view pose for the robot to drive to, which minimizes the expected search time, based on the information gathered. It also has to decide if an object was found and also if the room was fully searched. The difficulty of these problems, limited hardware resources and inaccurate sensor data and object detections made many simplifications to the problem necessary.




The search room tasks is the actual search within the room. It is the most complex task as an efficient execution of the object search has to be planned and executed. The concept to solve this task splits the problem into three smaller problems. The first is detecting if and where a searched object is found. The second is the generation of the best next robot pose. The third problem is the recognizing, if every part of the room was sufficiently searched. 

\subsubsection{Detection if Object was Found}

In course of the mapping an object map is created, which could be used here. However this is not the optimal solution as this map contains old information, which might be outdated, and due to efficiency reasons suboptimal parameters degrade the accuracy of this map. 

For the detection of objects the same principle, a grid map with object probabilities is used. As for the search only one object map has to be created as only one object type is of interest and neglectable uncertainty of the robot pose can be assumed as the room was already explored and a map exists. Therefore more computational power can be invested into the creation of this map leading to better results. 

An object is said to be found if the object probability in a cell exceeds a threshold parameter. The drawback of this solution is that not the actual position of the object is found but only a cell containing the object. This is noticeable for large objects, like a sofa.
