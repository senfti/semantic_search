\chapter{Evaluation}

In this chapter covers the evaluation of this active object system is covered. A detailed and meaningful quantitative evaluation of the system is impossible for multiple reasons. Firstly, the settings of a test case, like the environment, the searched object type, the object location and the starting pose, have a huge impact on the search time and the usefulness of intelligent behavior. Secondly, a lot of randomness, like measurement noise, randomness in the exact robot movement and timing of actions, is present, leading to very different results even with similar settings. Thirdly, test runs can easily take more than half an hour, so only a small number of test runs can be made and covering a wide variety of settings and having a large enough number of test runs to deal with the randomness is impossible. Therefore this evaluation is mostly done qualitatively by investigating representative runs and showing encountered special cases.

The evaluation is split into three parts. The first part covers the basic functionality, where the quality of the sensor data, the results of the computer vision, problems with the navigation and the general system stability are discussed. In the second part the quality of the generated maps is investigated. The last part evaluates the search performance, where the search performance within a single room and in larger indoor environments are tested.

Three environments were available for testing. Environment 1 is a home environment in a small flat, consisting of three rooms. Environment 2 is an atypical university environment consisting of some offices and the robotics-lab. Environment 3 is a part of an university floor with rooms of various types. A more detailed description of the environments can be found in section \ref{EvalMapping}.
