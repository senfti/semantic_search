\chapter{Evaluation}

In this chapter the active object search system described before is evaluated in practice. A detailed and meaningful quantitative evaluation of the complete system is impossible for multiple reasons. The test conditions, like the environment, the searched object type, the object location and the starting pose, have a huge impact on the search time and the utility of an intelligent behavior. The result is also influenced by random factors, like measurement noise, inaccuracies of the robot movement and timing of actions, which leads to very different results even with similar settings. Furthermore, test runs can easily take more than half an hour, so extensive testing is impossible. Therefore this evaluation is mostly done qualitatively by investigating representative runs and showing encountered special cases.

The evaluation is split into three parts. The first part covers the basic functionality, where the results of the computer vision and the performance of the navigation are discussed. In the second part the quality of the generated maps is investigated. The last part covers the object search, where the search performance within a single room and in larger indoor environments is tested.

Three environments were available for testing. Environment 1 is a home environment in a small flat, consisting of three rooms. Environment 2 is an atypical university environment consisting of some offices and the robotics-lab. Environment 3 is a part of an university floor with rooms of various types. A more detailed description of the environments can be found in Section \ref{EvalMapping}. The environments contain a lot of difficulties for the robot, like narrow passages, lots of obstacles and cluttered areas. Some minor modifications were made to the environment, like fencing off obstacles the robot cannot detect, neither with the LIDAR nor with the camera, and dangerous areas like stairs. However, this leaves still many difficulties for the robot, which it has to cope with.
