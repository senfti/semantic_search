\section{Search}

The results of a series of test runs are presented in the first part of this section, where the proposed intelligent active object search system and an uninformed system, which does not estimate likely object locations, are compared. In the second part of this section the behavior of the proposed system is evaluated in more detail based on individual runs. 

\subsection{Comparison between Intelligent and Uninformed Search}

\subsubsection{Settings}

Two changes were made to the proposed system to get an uninformed active object search system for comparison. Firstly, the detection of doorways is disabled in the uninformed system, so the room structure is ignored. Secondly, every cell that can contain the object, which are cells within the room and not marked as free, have the same probability of containing the object. This results in a coverage maximizing search strategy because the rest of the system is identical to the intelligent search system.

The test runs were done in the home environment. Eight different settings were chosen for the test runs, with four different searched objects and two different starting positions per object. Reliably detected objects were chosen to reduce the influence of the object detection and have more emphasis on the search planning. The objects were positioned in typical locations and a bit hidden, so they are not found during exploration and the robot has to search them. In Figure \ref{search_overview} the map of the environment is shown with marked room types, starting poses and object locations.

\subsubsection{Results}

In Figure \ref{timelengthavg} the average search times and trajectory lengths of the runs of the test series are shown. The intelligent search is significant faster than the uninformed search. The difference between the strategies is higher in the trajectory length. The reason is that the switching of high-level tasks and the peek tasks cost time, but no travel distance. The uninformed search has only one peek task and two high-level task switches, while the informed search has multiple of both. The time of about 5 seconds per high-level task switch and 20 seconds per peek task adds up. The variance in the search runs is high, therefore the search times and trajectory lengths are shown in more detail in Figure \ref{search times} and \ref{search length}. 

The uninformed search times vary a lot. On the one hand the remote and the mouse are seen from view poses, where a lot of area is covered, which is not the case for the other two objects. On the other hand the uninformed search heavily depends on where the exploration ends and where the search starts. When starting at starting pose 2 the exploration ends in room home2 and when starting at pose 2 the exploration ends in room home3. When starting at pose 1 the robot can go first either to room home2 or room home3, as shown in Figure \ref{explore_split}, though towards room home3 happens more often.
